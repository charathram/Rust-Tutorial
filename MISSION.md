# Mission: Rust

## Why
Become fluent enough in Rust to reach for it as a default tool — building command-line utilities and web backends that are fast, correct, and safe by construction, without a garbage collector or a class of memory bugs. The goal is real, shippable programs, not toy exercises.

## Success looks like
- Ship a non-trivial CLI tool in Rust (argument parsing, file/IO, error handling) end to end.
- Stand up a small web backend (HTTP server, routes, JSON) and run it.
- Read and reason about ownership, borrowing, and lifetimes well enough to fix compiler errors without guessing.
- Use `cargo`, the crate ecosystem, and `Result`-based error handling idiomatically.

## Constraints
- Open-ended timeline — no deadline. Optimise for durable understanding (storage strength) over speed.
- 30+ years of programming experience: comfortable in C, Java, Python, Go (and older languages). Lessons should leverage this — compare to what is already known and skip programming fundamentals.
- Prefers short, hands-on lessons grounded in real goals.

## Out of scope (for now)
- Embedded / no_std, WASM, async runtimes internals — revisit once the core model is solid.
- Macros authoring, unsafe Rust internals — later.
